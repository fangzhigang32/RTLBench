RTLBench : A Multi-Dimensional Benchmark Suite for Evaluating LLM-Generated RTL Code

Repository Paper

RTLBench is a benchmark suite for evaluating RTL code generated by large language models (LLMs). It contains 160 copyright-free design cases from textbooks and open-source projects, covering tasks from simple logic gates to complex state machines. Unlike prior benchmarks focusing only on syntax and functionality, RTLBench introduces five dimensions: syntax, functionality, lint compliance, readability, and coding style. It further employs an LLM-as-a-judge mechanism for subjective evaluation and Log2BetterRTL to iteratively refine code using EDA tool logs.

AutoEval Folder Structure

The AutoEval directory contains two key submodules:

1. EvalExistBenchmark

Evaluate the quality of reference RTL code in existing benchmarks.

‚Ä¢ Run Command:
python verilatorLint.py

2. RTLBench

Use RTLBench-Benchmark to evaluate LLM-generated RTL code.

‚Ä¢ Configuration:

Edit api.txt to set your API_KEY and BaseURL.

‚Ä¢ Run Command:
python main.py
‚Ä¢ Experimental Results:

The experimental results will be stored in the experiment/model_name/log folder.

  • summary_report.txt file stores the results of the first evaluation.
  • re_summary_report.txt file stores the results of Log2BetterRTL post-optimization evaluation.
